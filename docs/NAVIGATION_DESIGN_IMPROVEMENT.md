# 네비게이션 디자인 개선 계획

**작성일**: 2025-01-14  
**목표**: 아마추어적인 느낌을 제거하고 전문적이고 신뢰감 있는 네비게이션으로 개선

---

## 현재 문제점 분석

### 시각적 문제
1. **활성 상태가 불명확함**: 단순한 border-bottom만으로는 시각적 계층이 약함
2. **색상 대비 부족**: 비활성 탭이 너무 흐릿하게 보임 (#8A8A8E는 너무 연함)
3. **공간 활용 미흡**: 탭 간 간격과 패딩이 일관되지 않음
4. **전환 효과 부족**: 호버/활성 상태 전환이 부드럽지 않음
5. **모바일 탭 바가 평면적**: 깊이감과 계층 구조가 부족

### UX 문제
1. **현재 위치 인지 어려움**: 활성 탭이 명확하게 구분되지 않음
2. **터치 타겟 크기**: 모바일에서 터치 영역이 충분하지 않을 수 있음
3. **시각적 피드백 부족**: 클릭/터치 시 즉각적인 피드백이 약함

---

## 개선 방향 (15년차 웹디자이너 관점)

### 핵심 원칙
1. **명확한 시각적 계층**: 활성/비활성 상태가 한눈에 구분되어야 함
2. **부드러운 전환**: 모든 상태 변화는 자연스러운 애니메이션으로 처리
3. **일관된 디자인 언어**: design.md의 색상/타이포그래피 시스템 준수
4. **접근성 우선**: WCAG 2.1 AA 기준 준수 (색상 대비, 터치 타겟 크기)

---

## Phase 1: 데스크톱 네비게이션 개선 (1시간)

### 1.1 활성 탭 시각적 강화
**현재**: 단순한 border-bottom + 배경색
**개선**: 
- **배경색 강화**: `bg-[#3478F6]/10` → `bg-[#3478F6]/15` (더 명확한 배경)
- **하단 인디케이터**: border-bottom 두께를 `2px` → `3px`로 증가
- **텍스트 강조**: 활성 탭 텍스트를 `font-semibold` → `font-bold`로 변경
- **아이콘 강조**: 활성 탭 아이콘 색상도 Primary로 변경

### 1.2 비활성 탭 개선
**현재**: `#8A8A8E` (너무 연함)
**개선**:
- **색상 조정**: `#8A8A8E` → `#6E6E73` (약간 더 진하게, 가독성 향상)
- **호버 상태 강화**: 
  - 호버 시: `#1C1C1E` (기본 텍스트 색상)
  - 호버 배경: `bg-[#F2F2F7]` (미묘한 배경 변화)
  - 전환 효과: `transition-all duration-200 ease-in-out`

### 1.3 레이아웃 및 간격 개선
**현재**: 기본 간격
**개선**:
- **탭 간 간격**: `gap-1` → `gap-0` (탭이 하나의 그룹처럼 보이도록)
- **탭 패딩**: `px-4 py-3` → `px-6 py-4` (더 넉넉한 클릭 영역)
- **컨테이너 패딩**: `px-4 sm:px-6 lg:px-8` 유지 (헤더와 일관성)

### 1.4 시각적 계층 구조
```
[활성 탭]
- 배경: bg-[#3478F6]/15 (연한 파란색 배경)
- 텍스트: #3478F6 (Primary 색상)
- 아이콘: #3478F6
- 하단 인디케이터: 3px solid #3478F6
- 폰트: font-bold

[비활성 탭]
- 배경: transparent
- 텍스트: #6E6E73 (개선된 Neutral 색상)
- 아이콘: #6E6E73
- 하단 인디케이터: 없음
- 폰트: font-medium

[호버 상태]
- 배경: bg-[#F2F2F7] (미묘한 배경)
- 텍스트: #1C1C1E (기본 텍스트 색상)
- 전환: transition-all duration-200 ease-in-out
```

## Phase 2: 모바일 하단 탭 바 개선 (1시간)

### 2.1 활성 탭 시각적 강화
**현재**: 단순한 색상 변경
**개선**:
- **배경 강조**: 활성 탭에 `bg-[#3478F6]/10` 배경 추가
- **아이콘 크기**: 활성 탭 아이콘을 `w-6 h-6` → `w-7 h-7`로 약간 확대
- **텍스트 강조**: 활성 탭 텍스트를 `font-semibold` → `font-bold`로 변경
- **상단 인디케이터**: 활성 탭 상단에 `3px` 높이의 Primary 색상 바 추가

### 2.2 비활성 탭 개선
**현재**: `#8A8A8E` (너무 연함)
**개선**:
- **색상 조정**: `#8A8A8E` → `#6E6E73` (데스크톱과 동일)
- **터치 피드백**: `active:scale-95` 추가 (터치 시 약간 축소 효과)

### 2.3 탭 바 전체 스타일 개선
**현재**: 기본 스타일
**개선**:
- **배경 강화**: `bg-white/95` → `bg-white/98` (더 불투명하게)
- **그림자 개선**: `shadow-lg` → `shadow-xl` (더 명확한 깊이감)
- **상단 테두리**: `border-t border-gray-200/50` → `border-t border-gray-200/80` (더 명확한 구분)
- **탭 간 구분선**: 탭 사이에 미묘한 구분선 추가 (선택사항)

### 2.4 터치 타겟 크기 최적화
- **최소 터치 영역**: `h-16` 유지 (64px, WCAG 권장 44px 이상)
- **아이콘 + 텍스트 간격**: `gap-1` → `gap-1.5` (더 넉넉한 간격)
- **텍스트 크기**: `text-xs` 유지 (12px, 최소 크기 준수)

### 2.5 시각적 계층 구조
```
[활성 탭]
- 배경: bg-[#3478F6]/10
- 상단 인디케이터: 3px 높이, #3478F6
- 아이콘: w-7 h-7, #3478F6
- 텍스트: font-bold, #3478F6
- 크기: scale-110 (약간 확대)

[비활성 탭]
- 배경: transparent
- 아이콘: w-6 h-6, #6E6E73
- 텍스트: font-medium, #6E6E73
- 터치 피드백: active:scale-95
```

## Phase 3: 전환 효과 및 애니메이션 (30분)

### 3.1 상태 전환 애니메이션
- **탭 전환**: `transition-all duration-200 ease-in-out`
- **색상 전환**: `transition-colors duration-200`
- **크기 전환**: `transition-transform duration-200`

### 3.2 호버 효과
**데스크톱**:
- 배경색 전환: `hover:bg-[#F2F2F7]`
- 텍스트 색상 전환: `hover:text-[#1C1C1E]`
- 부드러운 전환: `duration-200 ease-in-out`

**모바일**:
- 터치 피드백: `active:scale-95` (터치 시 약간 축소)
- 색상 피드백: `active:bg-[#F2F2F7]` (터치 시 배경 변화)

### 3.3 활성 상태 표시 애니메이션
- 하단 인디케이터가 부드럽게 슬라이드되는 효과 (선택사항)
- 배경색이 페이드 인되는 효과

---

## Phase 4: 접근성 및 반응형 검증 (30분)

### 4.1 색상 대비 검증
- **활성 탭**: #3478F6 on #FFFFFF → 대비 비율 확인 (WCAG AA 이상)
- **비활성 탭**: #6E6E73 on #FFFFFF → 대비 비율 확인
- **호버 상태**: #1C1C1E on #F2F2F7 → 대비 비율 확인

### 4.2 터치 타겟 크기 검증
- 모바일: 최소 44px × 44px (WCAG 권장)
- 데스크톱: 최소 32px × 32px (마우스 클릭)

### 4.3 반응형 브레이크포인트 검증
- 모바일 (< 768px): 하단 탭 바
- 태블릿 (768px ~ 1023px): 상단 네비게이션 (데스크톱과 동일)
- 데스크톱 (≥ 1024px): 상단 네비게이션

---

## 구현 우선순위

### Priority 1 (즉시 개선) 🔴
1. 활성 탭 시각적 강화 (배경색, 인디케이터, 텍스트 강조)
2. 비활성 탭 색상 조정 (#8A8A8E → #6E6E73)
3. 호버 효과 개선

### Priority 2 (UX 개선) 🟡
1. 전환 효과 추가 (transition-all)
2. 모바일 탭 바 상단 인디케이터 추가
3. 터치 피드백 개선

### Priority 3 (세부 조정) 🟢
1. 탭 간 간격 및 패딩 최적화
2. 그림자 및 테두리 세부 조정
3. 애니메이션 세부 조정
