# 타임라인 기능 UI/UX 개선 작업 계획

## 목표
토스(TOSS) 스타일의 간단하고 명확한 UI/UX로 개선하여 중장년층이 쉽게 사용할 수 있도록 합니다.

## 핵심 원칙
1. **큰 버튼, 명확한 레이블**: 모든 클릭 가능한 요소는 최소 44x44px 터치 영역
2. **충분한 여백과 간격**: 요소 간 최소 16px 간격, 카드 간 24px 이상
3. **큰 폰트 크기**: 본문 최소 18px, 제목 24px 이상
4. **명확한 색상 구분**: 높은 대비율, 직관적인 색상 사용
5. **진행 상황 시각화**: 현재 어느 단계인지 명확히 표시
6. **초보자 안내**: 각 단계에서 무엇을 해야 하는지 명확히 안내

## 작업 목록

### 1단계: 폰트 크기 및 가독성 개선

#### 1.1 TimelineStepCard 컴포넌트 개선
**파일**: `components/timeline/TimelineStepCard.tsx`

**작업 내용**:
- 제목 폰트 크기 증가: `text-lg sm:text-xl` → `text-xl sm:text-2xl md:text-3xl`
- 설명 텍스트 크기 증가: `text-sm sm:text-base` → `text-base sm:text-lg`
- 배지 텍스트 크기 증가: `text-xs sm:text-sm` → `text-sm sm:text-base`
- 카드 패딩 증가: `p-4 sm:p-6` → `p-6 sm:p-8`
- 카드 간격 증가: `gap-4 sm:gap-6` → `gap-6 sm:gap-8`
- 단계 번호 원형 배지 크기 증가: `w-10 h-10 sm:w-12 sm:h-12` → `w-14 h-14 sm:w-16 sm:h-16`
- 단계 번호 폰트 크기 증가: `text-lg sm:text-xl` → `text-xl sm:text-2xl`
- 화살표 아이콘 크기 증가: `w-5 h-5 sm:w-6 sm:h-6` → `w-6 h-6 sm:w-7 sm:h-7`

**예상 효과**:
- 중장년층이 텍스트를 더 쉽게 읽을 수 있음
- 카드가 더 넓어져 터치하기 쉬움
- 시각적 계층 구조가 더 명확해짐

#### 1.2 TimelineStepContent 컴포넌트 개선
**파일**: `components/timeline/TimelineStepContent.tsx`

**작업 내용**:
- 탭 버튼 크기 증가:
  - 패딩: `px-3 py-2 sm:px-4` → `px-5 py-3 sm:px-6 sm:py-4`
  - 폰트: `text-xs sm:text-sm md:text-base` → `text-base sm:text-lg`
  - 아이콘: `w-4 h-4` → `w-5 h-5 sm:w-6 sm:h-6`
- 탭 버튼 간격 증가: `gap-2 sm:gap-3` → `gap-3 sm:gap-4`
- 콘텐츠 카드 패딩 증가: `p-4` → `p-5 sm:p-6`
- 콘텐츠 카드 간격 증가: `space-y-3` → `space-y-4 sm:space-y-5`
- 아이콘 크기 증가: `w-5 h-5` → `w-6 h-6 sm:w-7 sm:h-7`
- 이전/다음 버튼 크기 증가: 기본 버튼 크기에서 `size="lg"` 적용
- 버튼 텍스트 크기: `text-base sm:text-lg`로 명시

**예상 효과**:
- 탭 버튼이 더 크고 명확해져 클릭하기 쉬움
- 콘텐츠가 더 읽기 쉬움
- 버튼이 더 눈에 띄고 사용하기 편함

#### 1.3 페이지 헤더 개선
**파일**: `app/timeline/page.tsx`

**작업 내용**:
- 제목과 설명 간격 증가: `mt-2 sm:mt-3` → `mt-4 sm:mt-6`
- 전체 페이지 간격 증가: `space-y-6 sm:space-y-8 md:space-y-12` → `space-y-8 sm:space-y-10 md:space-y-16`

**예상 효과**:
- 페이지가 더 여유롭고 읽기 편함

### 3단계: 진행 상황 시각화 추가

#### 3.1 현재 단계 표시 기능 추가
**파일**: `components/timeline/TimelineStepCard.tsx`

**작업 내용**:
- 현재 단계인지 표시하는 prop 추가 (선택 사항)
- 현재 단계일 경우:
  - 배경색 변경: `bg-primary/10` 또는 `bg-primary/5` → `bg-primary/20`
  - 테두리 강조: `border-primary` 두께 증가
  - "현재 단계" 배지 추가
- 완료된 단계 표시 (선택 사항):
  - 체크 아이콘 추가

**예상 효과**:
- 사용자가 현재 어느 단계인지 명확히 알 수 있음
- 진행 상황을 시각적으로 파악 가능

**작업 내용**:

- "4단계 중 N단계 완료" 텍스트 표시

**예상 효과**:

- 토스 스타일의 명확한 진행 상황 표시

### 5단계: 색상 및 시각적 개선

#### 5.1 색상 대비 강화
**파일**: `components/timeline/TimelineStepCard.tsx`, `components/timeline/TimelineStepContent.tsx`

**작업 내용**:
- 텍스트 색상 대비 확인 및 강화:
  - 제목: `text-[#111827]` (거의 검정) 유지
  - 본문: `text-muted-foreground` → 더 진한 회색으로 변경 (가독성 향상)
- 배경색과 텍스트 색상 대비율 4.5:1 이상 보장 (WCAG AA 기준)
- 활성 상태 색상 강조:
  - 호버 시: `hover:bg-primary/5` → `hover:bg-primary/10`
  - 활성 탭: 배경색 더 진하게

**예상 효과**:
- 가독성 향상
- 시각적 피드백이 더 명확해짐

#### 5.2 아이콘 및 시각적 요소 개선

- 체크 아이콘, 경고 아이콘 등 시각적 요소 강조

**예상 효과**:
- 아이콘이 더 명확하게 보임

## 검증 방법

1. **시각적 검증**: 실제 화면에서 폰트 크기, 간격, 색상 확인
2. **터치 영역 검증**: 모바일에서 모든 버튼이 충분히 크고 클릭하기 쉬운지 확인
3. **가독성 검증**: 중장년층 테스터를 통한 실제 사용 테스트
4. **접근성 검증**: 키보드 네비게이션 및 스크린 리더 테스트
5. **반응형 검증**: 다양한 화면 크기에서 테스트

## 참고 사항

- 모든 변경 사항은 기존 기능을 유지하면서 UI/UX만 개선
- 토스 앱의 UI/UX를 참고하여 간단하고 명확한 디자인 유지
- 중장년층 사용자 테스트를 통한 피드백 반영 권장
