# 주요 기능 문서

이 디렉토리에는 프로젝트의 4가지 핵심 기능에 대한 상세 문서가 포함되어 있습니다.

## 기능 목록

### 1. [산재 진행 과정 (타임라인)](./timeline.md)
산재 신청부터 치료, 복귀까지의 전체 과정을 4단계로 나누어 시각적으로 표시하는 기능입니다.

**주요 특징:**
- 4단계 타임라인 구조
- 각 단계별 해야 할 일, 필요 서류, 주의사항 제공
- 초보자도 이해하기 쉬운 용어 사용
- 단계별 상세 페이지 제공

### 2. [병원 찾기](./hospitals.md)
산재 치료기관(병원, 약국, 재활기관, 직업훈련기관)을 지도와 목록으로 검색하는 기능입니다.

**주요 특징:**
- Naver Maps 기반 인터랙티브 지도
- 위치 기반 반경 검색 (5, 10, 15, 30km)
- 지역 선택 검색 (시/도, 시/군/구, 읍/면/동)
- 기관별 필터링 (병원, 약국, 재활, 직업훈련)
- 즐겨찾기 기능

### 3. [서류 안내](./documents.md)
산재 신청 및 치료 과정에서 필요한 8개 주요 서류에 대한 상세 안내를 제공하는 기능입니다.

**주요 특징:**
- 8개 핵심 서류 안내
- AI 기반 서류 작성 가이드 요약
- 서류별 맞춤 상담 챗봇
- 공식 양식 다운로드 링크

### 4. [산재 상담 챗봇](./chatbot.md)
n8n RAG 기술을 활용한 산업재해 전문 상담 챗봇 기능입니다.

**주요 특징:**
- 자연어 질문 및 답변
- RAG 기반 정확한 정보 검색
- 사용자별 세션 관리
- 마크다운 형식 답변 지원

## 공통 특징

### 접근 권한
- **공개 기능**: 타임라인, 병원 찾기, 서류 안내 (로그인 불필요)
- **인증 필요**: 챗봇, 즐겨찾기 (Clerk 인증 필요)

### 기술 스택
- **프레임워크**: Next.js 15 (App Router)
- **인증**: Clerk
- **데이터베이스**: Supabase (PostgreSQL)
- **지도**: Naver Maps API
- **AI**: Google Gemini API, n8n RAG 워크플로우
- **스타일링**: Tailwind CSS v4

### 디자인 원칙
- **초보자 친화적**: 복잡한 용어를 쉬운 표현으로 변경
- **고령자 친화적**: 큰 폰트, 명확한 버튼, 충분한 여백
- **반응형 디자인**: 모바일과 데스크톱 모두 최적화
- **일관된 디자인**: 시그니처 초록색, 통일된 UI 컴포넌트

## 문서 구조

각 기능 문서는 다음 섹션으로 구성되어 있습니다:

1. **개요**: 기능의 목적과 핵심 가치
2. **주요 기능**: 기능별 상세 설명
3. **기술 구성**: 페이지 구조, 컴포넌트, API, 데이터 구조
4. **데이터 흐름**: 기능 동작 과정
5. **접근 권한**: 인증 및 권한 요구사항
6. **사용자 경험**: UI/UX 특징 및 반응형 디자인
7. **주요 파일 경로**: 관련 파일 위치
8. **환경 변수**: 필요한 환경 변수
9. **향후 개선 가능 사항**: 추가 기능 아이디어
