# 병원 찾기 화면 재디자인 계획

**작성일**: 2025-01-14  
**목표**: 화면을 더 심플하고 심미적으로 개선하여 사용자 경험 향상

---

## 현재 문제점 분석

### 레이아웃 문제
1. **불필요한 제목**: "병원 찾기"라는 제목이 중복됨 (헤더 네비게이션에 이미 있음)
2. **여백 과다**: `py-8`로 인한 상하 여백이 과도함
3. **설명 텍스트**: "주변 산재 지정 의료기관을 찾아보세요."가 불필요할 수 있음

### 필터 UI 문제
1. **라벨 텍스트**: "검색 반경 선택", "기관 유형 선택" 라벨이 시각적 부담
2. **버튼 스타일**: 현재 버튼이 너무 크고 눈에 띄어 복잡해 보임
3. **숫자 표시**: "재활스포츠기관 (24개)"가 불필요한 정보 과부하

---

## 개선 방향 (15년차 웹디자이너 관점)

### 핵심 원칙
1. **Minimalism (미니멀리즘)**: 불필요한 요소 제거, 핵심 기능에 집중
2. **Visual Hierarchy (시각적 계층)**: 중요한 요소가 자연스럽게 눈에 들어오도록
3. **Elegant Simplicity (우아한 단순함)**: 심플하면서도 세련된 디자인

---

## Phase 1: 레이아웃 재구성 (30분)

### 1.1 제목 및 설명 제거
**현재**: 
- `<h1>병원 찾기</h1>` 제거
- `<p>주변 산재 지정 의료기관을 찾아보세요.</p>` 제거

**이유**: 
- 헤더 네비게이션에 이미 "병원 찾기" 메뉴가 있음
- 불필요한 설명은 사용자의 인지적 부담을 증가시킴

### 1.2 여백 최적화
**현재**: `py-8` (상하 32px)
**개선**: `pt-4 pb-2` (상 16px, 하 8px)
- 헤더 바로 아래에 필터가 오도록 여백 최소화
- 지도와 목록이 더 넓게 보이도록

### 1.3 컨테이너 구조 개선
**현재**: `container mx-auto px-4 py-8`
**개선**: `container mx-auto px-4 pt-4 pb-2`
- 상단 여백 최소화
- 전체적으로 위로 올라가도록

---

## Phase 2: 필터 UI 심플화 (1시간)

### 2.1 검색 반경 선택 개선
**현재**: 
- 라벨: "검색 반경 선택"
- 버튼: 큰 버튼, 배경색, 그림자

**개선**:
- 라벨 제거 (버튼 자체가 설명)
- 버튼 스타일:
  - 활성: `text-[#3478F6] font-semibold` (텍스트만 강조)
  - 비활성: `text-[#8A8A8E]` (회색 텍스트)
  - 호버: `hover:text-[#1C1C1E]` (진한 회색)
  - 배경색 제거, 그림자 제거
  - 간격: `gap-4` (더 넓은 간격)
  - 크기: `text-sm` (작은 텍스트)

### 2.2 기관 유형 선택 개선
**현재**: 
- 라벨: "기관 유형 선택"
- 버튼: 큰 버튼, 배경색, 그림자

**개선**:
- 라벨 제거
- 버튼 스타일:
  - 활성: 각 타입별 색상 텍스트 (병원: #3478F6, 약국: #34C759, 직업훈련기관: #9333EA)
  - 비활성: `text-[#8A8A8E]`
  - 호버: `hover:text-[#1C1C1E]`
  - 배경색 제거, 그림자 제거
  - 간격: `gap-4`
  - 크기: `text-sm`

### 2.3 숫자 제거
**현재**: "재활스포츠기관 (24개)"
**개선**: "재활스포츠기관"
- 불필요한 숫자 정보 제거
- 더 깔끔한 UI

---

## Phase 3: 레이아웃 통합 (30분)

### 3.1 필터 그룹 통합
**현재**: 두 개의 별도 섹션
**개선**: 하나의 통합된 필터 그룹
- 반경 선택과 기관 유형을 하나의 행으로 배치
- 구분선 또는 간격으로 시각적 구분

### 3.2 정보 텍스트 위치 조정
**현재**: 필터 위에 위치
**개선**: 필터 아래 또는 제거
- "지도 중심 기준 반경 5km 내 병원만 표시됩니다..." 텍스트
- 더 작은 크기로, 덜 눈에 띄게
