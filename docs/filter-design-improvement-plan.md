# 필터 UI 디자인 개선 계획

## 현재 문제점 분석

### 1. 시각적 복잡성
- **문제**: 여러 섹션이 세로로 나열되어 정보 과부하
- **증상**: 
  - 각 섹션마다 초록색 바 + 제목 + 버튼 그룹이 반복됨
  - 4-5개의 섹션이 연속으로 나열되어 압도적
  - 버튼들이 너무 많아서 선택이 어려움

### 2. 일관성 부족
- **문제**: 기본 필터와 고급 필터의 스타일이 다름
- **증상**:
  - 기본 필터(전체, 병원, 약국): 텍스트만, 배경 없음
  - 고급 필터 버튼: 활성 시 초록 배경, 비활성 시 흰색 배경 + 테두리
  - 같은 기능인데 다른 스타일로 표현됨

### 3. 불필요한 시각적 요소
- **문제**: 과도한 시각적 구분 요소
- **증상**:
  - 각 섹션마다 초록색 세로 바 (`w-1 h-4`)
  - 섹션 제목이 너무 강조됨 (`text-[16px] font-semibold`)
  - 구분선이 너무 많음 (`border-t`)

### 4. 정보 밀도 과다
- **문제**: 한 화면에 너무 많은 선택지
- **증상**:
  - 진료과목 6개 + 더보기 버튼
  - 검색 반경 4개 옵션
  - 재활기관 2개 옵션
  - 모든 것이 동시에 노출됨

## 개선 목표

### 핵심 원칙
1. **심플함 (Simplicity)**: 필수 정보만 표시
2. **전문성 (Professionalism)**: 일관된 디자인 언어
3. **명확성 (Clarity)**: 한눈에 이해 가능한 구조

## 개선 방안

### Phase 1: 필터 구조 단순화

#### 1.1 섹션 제목 제거
- **현재**: 각 섹션마다 "검색 방식", "검색 반경", "재활기관", "진료과목" 제목
- **개선**: 제목 제거, 버튼 자체가 설명
- **이유**: 버튼 텍스트만으로도 충분히 이해 가능

#### 1.2 초록색 바 제거
- **현재**: 각 섹션 제목 옆에 `w-1 h-4 bg-[#2F6E4F]` 바
- **개선**: 완전 제거
- **이유**: 불필요한 시각적 요소, 심플함 저해

#### 1.3 구분선 최소화
- **현재**: 각 섹션마다 `border-t border-[#E4E7E7]`
- **개선**: 섹션 간 간격만으로 구분 (`space-y-4`)
- **이유**: 구분선이 많으면 복잡해 보임

### Phase 2: 버튼 스타일 통일

#### 2.1 모든 버튼을 텍스트 기반으로 통일
- **현재**: 
  - 활성: 초록 배경 + 흰색 텍스트
  - 비활성: 흰색 배경 + 회색 테두리 + 회색 텍스트
- **개선**:
  - 활성: 초록색 텍스트 (`text-[#2F6E4F]`) + `font-semibold`
  - 비활성: 회색 텍스트 (`text-[#555555]`) + `font-medium`
  - 배경색 완전 제거
  - 테두리 제거
- **이유**: 
  - 일관성 확보
  - 기본 필터와 동일한 스타일
  - 더 깔끔하고 현대적인 느낌

#### 2.2 호버 효과 최소화
- **현재**: 호버 시 테두리 색상 변경
- **개선**: 호버 시 텍스트 색상만 약간 진하게 (`hover:text-[#1C1C1E]`)
- **이유**: 불필요한 시각적 변화 제거

### Phase 3: 레이아웃 최적화

#### 3.1 카드 배경 제거
- **현재**: `bg-white rounded-xl border border-[#E4E7E7] p-6`
- **개선**: 배경 없음, 단순한 구분선만 (`border-b border-[#E4E7E7] pb-4`)
- **이유**: 카드 형태가 오히려 복잡해 보임

#### 3.2 간격 조정
- **현재**: `space-y-6` (24px)
- **개선**: `space-y-4` (16px)
- **이유**: 더 컴팩트하고 효율적

#### 3.3 버튼 그룹 간격
- **현재**: `gap-2` (8px)
- **개선**: `gap-3` (12px)
- **이유**: 버튼이 너무 붙어있어서 답답함

### Phase 4: 정보 계층 구조 개선

#### 4.1 진료과목 필터 간소화
- **현재**: 6개 버튼 + 더보기 드롭다운
- **개선**: 
  - 기본 4개만 표시 (정형외과, 치과, 신경외과, 외과)
  - 나머지는 "더보기" 클릭 시 드롭다운
  - 선택된 개수 배지 제거 (너무 복잡함)

#### 4.2 검색 반경 통합
- **현재**: 별도 섹션으로 분리
- **개선**: 검색 방식 버튼 옆에 작은 텍스트로 표시
  - 예: "내 위치 주변 · 5km"
  - 또는 검색 방식 선택 시 자동으로 반경 선택 UI 표시

### Phase 5: 최종 디자인 스펙

#### 5.1 컨테이너
```css
/* 카드 제거, 단순 구분선 */
border-b border-[#E4E7E7] pb-4 mb-4
```

#### 5.2 버튼 스타일
```css
/* 활성 */
text-[#2F6E4F] font-semibold

/* 비활성 */
text-[#555555] font-medium hover:text-[#1C1C1E]

/* 공통 */
px-3 py-1.5 text-sm transition-colors
```

#### 5.3 섹션 간격
```css
space-y-4 /* 16px */
```

#### 5.4 버튼 그룹 간격
```css
gap-3 /* 12px */
```

## 예상 결과

### Before (현재)
- 복잡한 카드 구조
- 여러 섹션 제목과 초록색 바
- 혼재된 버튼 스타일
- 과도한 구분선

### After (개선 후)
- 단순한 구분선 구조
- 제목 없는 깔끔한 레이아웃
- 통일된 텍스트 기반 버튼
- 최소한의 시각적 요소

## 구현 우선순위

1. **High Priority**: 버튼 스타일 통일 (Phase 2)
2. **High Priority**: 섹션 제목 및 초록색 바 제거 (Phase 1.1, 1.2)
3. **Medium Priority**: 카드 배경 제거 (Phase 3.1)
4. **Medium Priority**: 간격 조정 (Phase 3.2, 3.3)
5. **Low Priority**: 진료과목 필터 간소화 (Phase 4.1)
