# 🌟 주요 기능 가이드 (Features Guide)

ReworkCare가 제공하는 핵심 기능들의 작동 원리와 관리 방법을 설명합니다.

## 1. 🤖 AI 산재 상담 챗봇

사용자가 궁금한 점을 물어보면 AI가 산재 관련 법령과 문서를 참고하여 답변해줍니다.

- **사용 기술**: Google Gemini Pro, Google File Search API (RAG)
- **작동 방식**:
  1.  공단 자료(PDF)를 Google Cloud에 미리 업로드해둡니다. (Vector Store)
  2.  사용자 질문이 들어오면 관련 내용을 PDF에서 찾습니다.
  3.  찾은 내용 + 질문을 AI에게 보내서 답변을 만듭니다.
- **관리 포인트**:
  - 새로운 산재 정책이 나오면 PDF 파일을 업데이트하고 재업로드해야 합니다.
  - `docs/google_file_search_implementation.md`에 상세한 업로드 방법이 있습니다.

---

## 2. 🏥 내 주변 산재 병원 찾기

사용자의 현재 위치나 선택한 지역을 기준으로 산재 지정 의료기관을 지도에 표시합니다.

- **사용 기술**: Naver Maps API, Supabase(좌표 데이터)
- **작동 방식**:
  1.  브라우저에서 사용자 위치(GPS) 권한을 요청합니다.
  2.  해당 좌표(위도/경도) 주변에 있는 병원들을 DB에서 검색합니다.
  3.  Naver 지도 위에 마커(핀)로 표시합니다.
- **관리 포인트**:
  - 병원이 폐업하거나 이전하면 DB(`hospitals` 테이블)를 수정해야 합니다.
  - 지도 API 사용량이 너무 많으면 비용이 발생할 수 있으니 Naver Cloud Console을 가끔 확인하세요.

---

## 3. 📝 산재 서류 가이드

복잡한 산재 신청 서류들을 단계별로 쉽게 설명해줍니다.

- **기능**:
  - 서류별 제출 시기 및 대상 안내
  - 작성 예시 이미지 제공
  - 빈 양식 다운로드
- **데이터 위치**: 코드 내 `lib/data/documents.ts` 파일에 텍스트가 저장되어 있습니다.
- **수정 방법**: 내용을 바꾸려면 위 파일을 열어서 텍스트를 고치고 배포하면 됩니다.

---

## 4. 📊 맞춤형 대시보드

사용자의 산재 진행 단계(요양 중, 휴업 중 등)에 따라 필요한 정보를 모아서 보여줍니다.

- **주요 위젯**:
  - **진행 상태**: 현재 단계 표시
  - **추천 정보**: 상황에 맞는 유튜브 영상, 정책 뉴스 추천
  - **급여 계산기**: 예상 보상금 모의 계산
- **특이사항**: 로그인하지 않은 게스트(Guest)에게는 '서울 시청' 기준의 예시 데이터를 보여줍니다.

---

## 5. 🗣️ 커뮤니티

사용자들이 서로 정보를 공유하고 위로할 수 있는 소통 공간입니다.

- **기능**: 게시글 작성/수정/삭제, 댓글, 조회수, 좋아요
- **정책**: 비방이나 욕설 등 부적절한 게시글은 관리자(Admin)가 DB에서 직접 삭제해야 합니다.

---

## 6. 🔐 회원가입 및 로그인 (Auth)

- **제공 방식**: 카카오톡 로그인, 구글 로그인, 이메일 가입
- **관리**: Clerk 대시보드에서 회원 목록을 보거나 특정 회원을 차단할 수 있습니다.
- **주의**: Clerk에서 회원을 삭제해도 Supabase DB에는 데이터가 남을 수 있습니다. 완전 삭제를 원하면 양쪽 다 지워야 합니다.
