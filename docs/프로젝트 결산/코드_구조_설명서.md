# 🏗️ 코드 구조 및 파일 설명 (Code Structure)

어떤 파일이 무슨 역할을 하는지 알면 수정이 쉬워집니다. 중요한 파일들만 골라서 설명합니다.

## 1. 폴더 구조 (Directory Map)

```
hospital_data/
├── app/                    # 🌐 웹페이지 (URL과 매칭됨)
│   ├── page.tsx            # 메인 홈페이지 (www.reworkcare.com/)
│   ├── layout.tsx          # 모든 페이지의 공통 테두리 (헤더, 푸터 등)
│   ├── dashboard/          # 대시보드 (/dashboard)
│   ├── community/          # 커뮤니티 (/community)
│   └── documents/          # 서류 가이드 (/documents)
│
├── components/             # 🧩 조립 부품 (UI 컴포넌트)
│   ├── ui/                 # 버튼, 입력창 같은 작은 부품들
│   ├── dashboard/          # 대시보드에만 쓰이는 부품들
│   └── Chatbot.tsx         # AI 챗봇 컴포넌트
│
├── lib/                    # ⚙️ 기능 로직 (눈에 안 보이는 것들)
│   ├── supabase/           # 데이터베이스 연결 코드
│   ├── data/               # 고정된 데이터 (서류 목록 등)
│   └── utils.ts            # 자주 쓰는 편리한 도구들
│
└── public/                 # 🖼️ 이미지 및 파일
    ├── images/             # 로고, 배너 이미지
    └── fonts/              # 글꼴 파일
```

---

## 2. 자주 수정하는 파일 위치 (Hotspots)

### 🎨 디자인/글자 수정

- **메인 홈페이지 문구**: `app/page.tsx`
- **상단 메뉴바(헤더)**: `components/Navbar.tsx`
- **사이트 전체 색상/폰트**: `app/globals.css` 또는 `tailwind.config.ts`

### 📋 데이터 수정

- **서류 가이드 내용**: `lib/data/documents.ts`
  - 이 파일에 있는 텍스트만 고치면 홈페이지 내용이 바뀝니다.
- **산재 병원 데이터**: (코드가 아니라) **Supabase 대시보드**에서 수정합니다.

### ⚙️ 설정 변경

- **사이트 이름/설명 (SEO)**: `app/layout.tsx`의 `metadata` 부분
- **API 주소/키**: `.env` 파일 (로컬) 또는 Vercel 환경변수

## 3. 핵심 파일 상세 설명

### `app/layout.tsx`

- **역할**: 집의 기둥 같은 파일입니다.
- **주의**: 여기서 `ClerkProvider`나 `html`, `body` 태그를 잘못 건드리면 사이트 전체가 깨집니다. 폰트나 SEO 메타태그를 바꿀 때만 조심스럽게 수정하세요.

### `lib/supabase/clerk-client.ts`

- **역할**: 로그인한 사용자가 자기 데이터만 볼 수 있게 해주는 보안관입니다.
- **경고**: 🛑 **절대 수정하지 마세요!** 이 파일이 고장나면 로그인이 안 되거나 보안 사고가 날 수 있습니다.

### `app/api/chat/route.ts` (또는 유사 경로)

- **역할**: 챗봇이 답변을 만드는 공장입니다.
- **내용**: 사용자 질문을 받아서 Google AI에게 보내고, 답을 받아오는 과정이 들어있습니다. 챗봇 답변 스타일(프롬프트)을 바꾸고 싶다면 여기서 수정합니다.
