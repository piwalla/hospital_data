# 🚀 퓨처 로드맵 (Future Roadmap)

현재 구현되지 않았지만, 향후 프로젝트 발전을 위해 **추가하면 좋은 기능들과 아이디어**를 정리한 문서입니다.  
시간이 날 때 하나씩 도전해보세요!

---

## Phase 2: 운영 효율화 및 데이터 고도화 (Next Step)

가장 우선순위가 높은 개선 사항들입니다.

- [ ] **관리자(Admin) 대시보드 강화**
  - **아이디어**: 현재 DB 직접 수정 방식에서 벗어나, 관리자 전용 페이지(`/admin`)에서 게시글 관리, 회원 관리를 할 수 있게 만듭니다.
  - **기술**: Retool 같은 로우코드 툴을 붙이거나, Next.js로 간단한 Admin 페이지 제작.

- [ ] **약국/병원 데이터 자동 최신화**
  - **아이디어**: 매번 CSV를 올리는 대신, 공공데이터포털 API를 연동하여 매일 밤 자동으로 데이터를 갱신합니다.
  - **기술**: Supabase Edge Functions + Cron Job 활용.

- [ ] **SEO (검색 엔진 최적화) 강화**
  - **아이디어**: 구글이나 네이버에서 "산재 병원" 검색 시 우리 사이트가 상단에 뜨도록 합니다.
  - **기술**: `sitemap.xml` 생성, 동적 메타 태그(`generateMetadata`) 적용.

---

## Phase 3: 사용자 경험 확장 (Expansion)

- [ ] **모바일 PWA (앱 설치)**
  - **아이디어**: 앱스토어 심사 없이 웹사이트를 앱처럼 설치해서 쓸 수 있게 만듭니다.
  - **기술**: `next-pwa` 라이브러리 설정, `manifest.json` 작성.

- [ ] **전문가 상담 매칭**
  - **아이디어**: 노무사나 손해사정사와 채팅으로 상담할 수 있는 유료/무료 기능을 붙입니다.
  - **기술**: 실시간 채팅(Supabase Realtime) 기능 확장.

- [ ] **커뮤니티 활성화 기능**
  - **아이디어**: '좋아요', '스크랩', '베스트 게시글' 기능을 추가하여 참여를 유도합니다.

---

## 💡 아이디어 백로그 (Idea Backlog)

아직 구체화되진 않았지만 재미있는 아이디어들입니다.

- **산재 계산기 고도화**: 휴업급여뿐만 아니라 장해급여 예상액까지 계산해주는 기능.
- **병원 후기 시스템**: 사용자가 다녀온 산재 병원에 별점과 후기를 남기는 기능. (단, 의료법 준수 주의)
- **Voice UI**: 타자를 치기 힘든 환자를 위해 음성으로 챗봇에게 물어보는 기능. (Web Speech API)
