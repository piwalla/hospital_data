# 🔧 문제 해결 가이드 (Troubleshooting)

운영 중에 자주 발생하는 문제와 해결 방법을 정리했습니다.

## 1. 챗봇이 "오류가 발생했습니다"라고 해요 🤖

**원인**

- Google AI API 사용량이 초과되었거나 키가 만료됨
- Supabase의 Vector Store 문서가 손상됨

**해결 방법**

1.  **API 키 확인**: `.env` 또는 Vercel 환경변수의 `GOOGLE_GENERATIVE_AI_API_KEY`가 맞는지 확인하고, 필요하면 재발급 받으세요.
2.  **로그 확인**: Vercel Logs에서 에러 메시지를 확인하세요. `429 Too Many Requests`는 사용량 초과입니다. 잠시 기다렸다가 다시 시도하세요.
3.  **AI에게 묻기**: "챗봇이 동작하지 않습니다. Vercel 로그에 [에러메시지]가 뜹니다."라고 물어보세요.

---

## 2. 지도에 병원이 하나도 안 나와요 🗺️

**원인**

- 네이버 지도 API 설정 문제
- 현재 위치 권한 거부
- 데이터베이스에 해당 지역 병원 데이터 없음

**해결 방법**

1.  **위치 권한**: 브라우저 주소창 왼쪽의 자물쇠 아이콘🔒을 눌러 '위치' 권한이 허용되어 있는지 확인하세요.
2.  **API 키**: `.env`에 `NEXT_PUBLIC_NAVER_CLIENT_ID`가 올바른지 확인하세요. 또한 `localhost:3000`이나 실제 도메인이 네이버 콘솔에 등록되어 있어야 합니다.
3.  **데이터 확인**: Supabase `hospitals` 테이블에 데이터가 들어있는지 확인하세요.

---

## 3. "로그인에 실패했습니다" 경고창이 떠요 🚫

**원인**

- 소셜 로그인 설정 오류
- 도메인 허용 목록 누락

**해결 방법**

1.  Clerk 대시보드에서 `User & Authentication` -> `Social Connections` 설정이 올바른지 확인하세요.
2.  개발 중(`localhost`)이라면 Clerk 대시보드의 `Development` 모드가 켜져 있는지 확인하세요.

---

## 4. 수정한 내용이 웹사이트에 반영이 안 돼요 🔄

**원인**

- 브라우저 캐시(Cache) 문제
- 배포 실패

**해결 방법**

1.  **새로고침**: `Ctrl + Shift + R` (윈도우) 또는 `Cmd + Shift + R` (맥)을 눌러 강력 새로고침을 하세요.
2.  **배포 상태 확인**: Vercel Dashboard에서 최근 배포가 `Ready` 상태인지, `Error` 상태인지 확인하세요. `Error`라면 로그를 보고 수정해야 합니다.

---

## 5. 엑셀(CSV) 파일 업로드가 안 돼요 📄

**원인**

- CSV 파일 형식이 맞지 않음
- 한글 인코딩 문제

**해결 방법**

1.  **형식 확인**: `name`, `address` 등 컬럼 이름(헤더)이 영어로 정확히 되어 있는지 확인하세요.
2.  **인코딩**: 엑셀에서 저장할 때 `CSV (UTF-8)` 형식으로 저장했는지 확인하세요. 일반 CSV는 한글이 깨질 수 있습니다.
3.  **가이드 참고**: `docs/CSV_IMPORT_GUIDE.md` 문서를 다시 읽어보세요.

---
