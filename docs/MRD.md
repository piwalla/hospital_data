# MRD: 산재환자 통합지원 플랫폼

---

## 1. 시장 개요

### 1.1 문제 정의

대한민국 산업재해 시장의 주요 문제점들:

- **산업재해 발생 규모**: 매년 약 **14만 명** 이상의 근로자가 산업재해를 경험
- **행정 절차 복잡성**: 산재 신청, 절차, 서류, 병원 연계 과정이 복잡하고 비효율적
- **심리적 지원 부재**: 심리·정서적 트라우마를 겪는 환자와 가족에 대한 체계적 지원 부족
- **정보 접근성**: 온라인상에 흩어진 정보로 인해 신뢰할 수 있는 **원스톱 플랫폼**의 필요성 증가

### 1.2 시장 기회

#### 디지털 전환 기회
- **디지털 복지 행정**: 정부의 전자민원 고도화 정책에 따른 자동화 수요 증가
- **심리 케어 시장 성장**: AI 기반 명상, 멘탈케어 서비스의 시장 확대

#### 비즈니스 기회
- **전문 서비스 제휴**: 노무사 및 병원 제휴 서비스 시장 성장
- **광고/어필리에이트**: 초기 수익원을 위한 다양한 마케팅 채널 활용 가능

## 2. 타겟 고객

### 타겟 세분화

| 구분 | 설명 |
|------|------|
| **1차 타겟** | 산재 피해 근로자 (산재 신청자, 재활·복귀 단계 근로자) |
| **2차 타겟** | 환자 가족, 보호자, 간병인 |
| **3차 타겟** | 노무사, 병원, 복지기관 등 제휴 가능 기관 |

### 주요 고객 페르소나

#### 1차 페르소나: 박OO (45세 남성, 현장 근로자)
- **상황**: 손목 부상으로 치료 중
- **페인포인트**: 복잡한 산재 절차로 인한 혼란과 스트레스
- **니즈**: 명확하고 단계별로 안내되는 신청 절차

#### 2차 페르소나: 이OO (38세 여성, 전업주부)
- **상황**: 남편이 산재로 입원 중
- **페인포인트**: 행정 절차, 병원 이동, 복지 정보 부족
- **니즈**: 종합적인 케어 가이드와 실질적인 지원 정보

#### 3차 페르소나: 김OO (노무사)
- **상황**: 산재 관련 클라이언트 확보를 위한 디지털 마케팅 채널 탐색
- **페인포인트**: 클라이언트 유치 경쟁 심화
- **니즈**: 전문성을 입증할 수 있는 플랫폼 제휴

## 3. 시장 규모 및 경쟁 분석

### 3.1 TAM / SAM / SOM 분석

| 구분 | 설명 | 규모 (추정치) |
|------|------|---------------|
| **TAM**<br/>(Total Addressable Market) | 전체 산업재해 근로자 + 가족 | 약 **60만 명** |
| **SAM**<br/>(Serviceable Addressable Market) | 온라인 정보/서비스 이용 가능층 | 약 **25만 명** |
| **SOM**<br/>(Serviceable Obtainable Market) | 초기 2년 내 확보 가능한 실사용자 | 약 **5만 명** |

### 3.2 경쟁 현황 분석

| 경쟁사 | 특징 | 한계점 |
|--------|------|--------|
| **근로복지공단 홈페이지** | 공공 정보 제공 | 사용자 친화적 UX 부족 |
| **네이버 블로그·카페** | 커뮤니티 중심 | 정보 신뢰성 낮음 |
| **법률사무소·노무사 사이트** | 전문 정보 제공 | 단발성 접근, 통합 서비스 부재 |
| **마인드카페, 토닥** | 심리 케어 앱 | 산재 맥락 맞춤형 아님 |

#### 시장 포지셔닝
- **근로복지공단**: 공식성이 강하지만 접근성이 낮음
- **민간 서비스**: 부분적 해결책만 제공, 통합 플랫폼 부재
- **우리 서비스**: 공식성 + 편의성 + 심리 케어의 균형

## 4. 고객 요구사항

### 요구사항 카테고리별 분석

| 구분 | 주요 요구사항 |
|------|---------------|
| **정보 접근성** | 서류, 절차, 병원 정보가 명확히 정리된 서비스 |
| **맞춤 지원** | 개인 상황에 맞는 절차 가이드 및 병원 추천 |
| **심리적 안정** | 명상, 치유 콘텐츠, 트라우마 완화 프로그램 |
| **신뢰성** | 공공·전문가 근거 기반의 정보 |
| **편의성** | 위치 기반 서비스, AI 챗봇 안내, 모바일 UX 최적화 |

### 상세 요구사항 우선순위

#### 🔥 **High Priority (필수)**
- **정보 접근성**: 복잡한 산재 절차를 단계별로 명확히 안내
- **신뢰성**: 공공기관 정보 기반의 검증된 콘텐츠 제공

#### 🟡 **Medium Priority (중요)**
- **편의성**: 모바일 최적화와 직관적인 UX
- **맞춤 지원**: 상황별 맞춤 가이드 제공

#### 🟢 **Low Priority (선택)**
- **심리적 안정**: 부가적인 웰빙 서비스로 발전 가능

## 5. 주요 기능 요구사항

### 기능 개발 로드맵

| 단계 | 타임라인 | 주요 기능 | 목표 |
|------|----------|-----------|------|
| **MVP** | 0~3개월 | 위치 기반 산재 병원·약국 지도, 서류 다운로드, 신청 절차 가이드, 애드센스/어필리에이트 수익화 | 기본 기능 검증 |
| **확장 1단계** | 3~6개월 | 명상 콘텐츠 AI 추천, 트라우마 치유 스토리, 사용자 커뮤니티 | 심리 케어 강화 |
| **확장 2단계** | 6~12개월 | RAG 기반 산재 규정 AI 상담, 제휴 노무사 연결, 가족 케어 매칭, 간병인 연결 | 전문 서비스 연계 |
| **확장 3단계** | 1년+ | 환자 복귀 지원 프로그램, 심리 안정 구독 모델, 오프라인 협약 병원 네트워크 | 생태계 구축 |

### MVP 핵심 기능 상세

#### 🗺️ **위치 기반 병원/약국 지도**
- 산재 지정 의료기관 위치 정보 제공
- 사용자 위치 기반 반경 검색
- 지도 위 마커 표시 및 상세 정보

#### 📄 **서류 다운로드 및 가이드**
- 주요 산재 신청 서류 제공
- 단계별 작성 가이드
- 공식 서식 다운로드

#### 📋 **신청 절차 안내**
- 산재 신청 전체 프로세스 안내
- 체크리스트 및 타임라인 제공
- 진행 상황 추적 기능

#### 💰 **수익화 기능**
- 애드센스 배너 광고
- 제휴 서비스 어필리에이트

## 6. 수익화 전략

### 단계별 수익 모델 발전

| 단계 | 수익 모델 | 설명 | 예상 수익 규모 |
|------|-----------|------|----------------|
| **초기**<br/>(0~6개월) | 애드센스, 어필리에이트 | 비상업적 정보 내에서 자연 노출 | 월 50~200만 원 |
| **1단계**<br/>(6~12개월) | 제휴 수익 | 노무사, 병원, 심리 콘텐츠 제휴 | 월 200~500만 원 |
| **2단계**<br/>(1~2년) | 구독형 모델 | 명상·심리 회복 콘텐츠 구독 | 월 500~1,000만 원 |
| **3단계**<br/>(2년+) | 서비스 커미션 | 간병인 연결, 병원 예약, 복귀 지원 프로그램 수수료 | 월 1,000만 원+ |

### 초기 수익화 원칙
- **비침해적 접근**: 사용자의 주요 정보 탐색을 방해하지 않는 선에서 수익화
- **가치 우선**: 먼저 사용자 가치를 입증한 후 수익화 모델 적용
- **다각화**: 단일 수익원에 의존하지 않고 포트폴리오 구축

## 7. 시장 진입 전략

### 4단계 진입 로드맵

#### 1️⃣ **초기 SEO 집중형 콘텐츠 마케팅**
- **콘텐츠 전략**: 산재 서류, 병원 정보, 실제 사례 중심의 고품질 콘텐츠 확보
- **목표**: 검색 엔진 상위 노출을 통한 자연스러운 트래픽 유입
- **주요 키워드**: "산재 신청 방법", "산재 지정 병원", "요양급여 신청"

#### 2️⃣ **네이버·구글 기반 트래픽 확보**
- **검색 최적화**: SEO/SMO 최적화로 검색 결과 상위권 확보
- **SNS 연계**: 네이버 블로그, 카페, 커뮤니티를 통한 바이럴 효과
- **리퍼럴 프로그램**: 사용자 추천을 통한 성장 동력 확보

#### 3️⃣ **신뢰 구축 및 브랜딩**
- **공식 협력**: 근로복지공단 정보 기반 콘텐츠 제공
- **전문가 제휴**: 노무사, 병원 로고 노출을 통한 신뢰성 강화
- **사용자 피드백**: 실제 사용자 후기와 개선 사례 공유

#### 4️⃣ **데이터 기반 확장**
- **사용자 분석**: 검색 패턴, 체류 시간, 이탈 지점 분석
- **개인화**: 데이터 기반 추천 서비스 강화
- **A/B 테스트**: 다양한 기능과 UI의 효과 측정
