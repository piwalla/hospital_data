## 1. 코어 디자인 철학

### 불안을 덜어주는 신뢰감 있는 길잡이

모든 디자인 요소는 사용자가 정신적으로 지치고 불안한 상태라는 것을 전제합니다. 화려함이나 복잡함을 철저히 배제하고, 마치 병원의 친절한 코디네이터가 옆에서 차분하게 안내해주는 듯한 경험을 제공하는 것을 목표로 합니다.

#### 핵심 원칙
- **Calm (차분함)**: 시각적 자극을 최소화하여 사용자의 인지적 부담을 줄입니다.
- **Clarity (명료함)**: 다음에 무엇을 해야 할지, 이 정보가 무엇을 의미하는지 즉시 알 수 있도록 합니다.
- **Guidance (길잡이)**: 사용자를 올바른 방향으로 자연스럽게 이끄는 시각적 단서를 제공합니다.

## 2. 색상 팔레트 (Color Palette)

신뢰와 안정을 주는 색상을 중심으로, 명확한 행동 유도를 위한 포인트 색상을 사용합니다. 2025년 기준 Modern SaaS 스타일을 반영하여 채도를 낮추고 중립 배경을 강화했습니다.

### Primary (주 색상)
- **색상**: `#2F6E4F` (톤다운 초록) - 기존 `#2E7D32`에서 채도를 낮춘 현대적인 초록색
- **사용처**: 핵심 버튼, 활성화된 메뉴, 링크, 주요 강조 요소 등
- **의미**: 신뢰, 안정, 치유, 성장. 사용자가 서비스를 믿고 의지할 수 있도록 하는 의료/케어 서비스에 적합한 색상입니다.
- **선정 이유**: 톤다운된 초록색은 자연, 건강, 성장을 상징하면서도 2010년대 웹 느낌을 벗어나 현대적이고 전문적인 인상을 전달합니다.

### Secondary (보조 색상)
- **색상**: `#61C48C` (부드러운 서브 초록) - 기존 `#34C759`에서 더 부드럽게 조정
- **사용처**: 긍정적 알림, 완료 상태, 유용한 팁 아이콘 등
- **의미**: 치유, 희망, 긍정. 사용자에게 정서적 지지를 전달합니다.

### Neutral (중립 색상)
- **배경**: `#F7F9F8` (녹색 기운의 연그레이) - 기존 `#F2F2F7`에서 녹색 기운을 더한 배경색
- **컨테이너**: `#FFFFFF` (흰색) - 카드, 모달 배경
- **기본 텍스트**: `#1C1C1E` (진한 회색) - 순흑색보다 부드러운 인상을 줍니다.
- **보조 텍스트**: `#555555` (중간 회색) - 기존 `#8A8A8E`에서 더 진하게 조정하여 가독성 향상
- **테두리**: `#E4E7E7` / `#D0D5D4` (연한 회색) - 테두리, 구분선에 사용

### Alert (주의 색상)
- **색상**: `#FF9500` (부드러운 주황색)
- **사용처**: 법적 고지(Disclaimer) 등 주의가 필요한 정보에 사용합니다.
- **주의**: 공격적인 빨간색은 사용자의 불안감을 증폭시킬 수 있어 지양합니다.

## 3. 타이포그래피 (Typography)

가독성을 최우선으로 고려하여, 모든 연령대가 쉽게 읽을 수 있는 폰트와 계층 구조를 사용합니다.

### 폰트
- **브랜드 폰트**: Paperozi (파페로지)
  - **사용처**: 로고, 브랜드명 **전용** (UI 전반에는 사용하지 않음)
  - **폰트 웨이트**: 100-900 (Thin ~ Black)
  - **선정이유**: 전문적이고 신뢰감 있는 브랜드 아이덴티티 표현
  - **폰트 소스**: [Project Noonnu](https://github.com/projectnoonnu/2408-3)
  - **주의**: UI 전반에 사용하면 전문성이 떨어지므로 로고/브랜드명에만 제한적으로 사용합니다.
  
- **본문 폰트**: Pretendard (프리텐더드)
  - **사용처**: 본문, 일반 텍스트, UI 요소 **전체**
  - **선정이유**: 본문 가독성이 매우 뛰어나고, 다양한 굵기를 지원하여 정보의 위계를 명확히 표현하기에 최적입니다.
  - **일관성**: UI 전반을 Pretendard로 통일하여 안정감과 전문성을 확보합니다.

### 사이즈 및 계층
- **제목 (H1)**: `30px` (700, Bold) - letter-spacing: -0.5px 추가하여 대비 강화
- **소제목 (H2)**: `22px` (600, SemiBold) - 각 섹션을 구분
- **소제목 (H3)**: `18px` (600, SemiBold) - 하위 섹션 구분
- **본문 (Body)**: `16px` (400, Regular) - 기존 17px에서 조정하여 현대적인 느낌
- **부가 설명 (Caption)**: `14px` (400, Regular) - 보조 정보, 출처 표기

## 4. 레이아웃 및 핵심 컴포넌트

Cursor와 같은 툴로 빠르게 구현할 수 있는 모듈식 컴포넌트 시스템을 제안합니다.

### 전체 구조: 반응형 네비게이션

**웹 서비스 특성**: 데스크톱과 모바일 모두에서 최적화된 경험 제공

#### 모바일 (< 768px): 하단 탭 바 (Bottom Tab Bar)
- **구성**: 두 개의 명확한 탭 `[병원 찾기]` 와 `[서류 안내]`
- **이유**: 스마트폰 사용자가 가장 익숙한 구조로, 핵심 기능을 언제든 즉시 전환할 수 있습니다.

#### 태블릿/데스크톱 (≥ 768px): 상단 네비게이션 또는 사이드바
- **구성**: 상단 헤더에 탭 메뉴 또는 좌측 사이드바
- **이유**: 넓은 화면을 활용하여 더 많은 정보를 한눈에 볼 수 있도록 합니다.
- **레이아웃**: 2-3 컬럼 그리드로 지도와 목록을 동시에 표시 가능

### 컴포넌트 1: 정보 카드 (Info Card)
- **특징**: 모든 정보를 독립된 카드 형태로 제공
- **스타일 규격**:
  - 배경: `#FFFFFF` (흰색)
  - 테두리: `1px solid #E4E7E7`
  - Border-radius: `12px`
  - Padding: `24px`
  - 그림자: 제거 (Border 중심 구조)
- **이유**: 정보의 단위를 명확히 구분하여 인지적 부담을 줄이고, 일관된 UI를 유지합니다. 그림자 대신 Border로 구분하여 현대적인 느낌을 줍니다.

### 컴포넌트 2: 단계별 아코디언 (Step-by-Step Accordion)
- **사용처**: 서류 안내와 같이 복잡한 절차를 단계별로 보여줄 때
- **특징**: 사용자는 현재 단계에만 집중할 수 있습니다.
- **개선사항**: 카드 형태로 개선, 열림 상태 배경 `#F7F9F8`로 변경

### 레이아웃 규칙
- **섹션 여백**: 상/하 48px 고정
- **카드 패딩**: 24px
- **본문 최대 너비**: 680px (본문 중심 페이지)
- **간격 규칙**: 12px 단위로 통일
- **그림자**: 제거 → Border 중심 구조로 전환
- **아이콘 스타일**: 모두 Outline 스타일로 통일 (stroke-width: 1.75)

### 버튼 스타일 가이드
- **Primary CTA**: `#2F6E4F` (초록) - 주요 액션 버튼
- **Secondary CTA**: Border `#D0D5D4` + 텍스트 진회색 - 보조 액션 버튼
- **Tertiary CTA**: 텍스트 버튼 `#4A4A4A` - 부가 액션 버튼
- **그림자**: 모든 버튼에서 제거

## 5. 화면별 디자인 컨셉 (Screen-by-Screen)


### 핵심 구조: 하단 탭 바 (Bottom Tab Navigation)

스마트폰 사용자가 가장 익숙한 구조입니다. 앱의 핵심 기능 두 가지를 언제든 즉시 이동할 수 있어 안정감을 줍니다.

#### 탭 구성
- **[아이콘 + 텍스트]** 형태로 구성하여 명확성을 높입니다.
- **[병원 찾기] | [서류 안내]** - 두 개의 탭으로 극도로 단순화합니다.

```ascii
+-----------------------------------------+
|                                         |
|                                         |
|            (각 화면의 내용)               |
|                                         |
|                                         |
+-----------------------------------------+
|  📍 병원 찾기      📄 서류 안내         |
+-----------------------------------------+
```

## 화면 1: 홈 / 병원 찾기 (첫 실행 화면)

**목표**: 사용자가 웹사이트를 열자마자, 아무런 설명 없이도 내 주변 병원을 바로 찾을 수 있게 한다.

**디자인 철학**: 
- **Minimalism (미니멀리즘)**: 불필요한 제목, 설명, 라벨 제거
- **Elegant Simplicity (우아한 단순함)**: 텍스트 기반의 심플한 필터 UI
- **Visual Hierarchy (시각적 계층)**: 핵심 기능(지도, 목록)에 집중

**반응형 레이아웃**:
- **모바일**: 세로 레이아웃 (지도가 상단, 목록이 하단)
- **데스크톱**: 가로 레이아웃 (필터/목록이 좌측, 지도가 우측 또는 전체 화면)

```ascii
+-----------------------------------------+
| [1] 리워크케어                          |
+-----------------------------------------+
| [2] [ 정형외과 ] [ 신경외과 ] [ 재활의학과 ] + |
+-----------------------------------------+
|                                         |
|                                         |
|                  [3]                    |
|                지도 영역                  |
|          (병원 위치 핀 표시)              |
|                                         |
|                                         |
+-----------------------------------------+
| [4] 내 주변 산재 지정 병원 (n개)      ^   |
| +-------------------------------------+ |
| | [병원 A] | 병원명 / 주소 / 1.2km   > | |
| +-------------------------------------+ |
| | [병원 B] | 병원명 / 주소 / 1.8km   > | |
| +-------------------------------------+ |
+-----------------------------------------+
```

#### 주요 요소 설명

##### 헤더 (Header)
- **브랜드명**: "리워크케어" - Paperozi 폰트 사용 (font-weight: 700-800)
- **스타일**: 
  - 색상: `#1C1C1E` (Neutral 텍스트) 또는 `#2E7D32` (Primary)
  - 크기: text-2xl 또는 text-3xl
  - 배경: `#FFFFFF` (흰색) 또는 `#F2F2F7` (Neutral 배경)
  - 그림자: subtle shadow (shadow-sm)
  - 호버 효과: Primary 색상으로 전환
- **레이아웃**: 
  - 최대 너비: max-w-7xl
  - 중앙 정렬
  - 적절한 패딩 및 간격
  - sticky positioning (스크롤 시 상단 고정)
- **반응형**: 모바일에서도 명확하게 보이도록 크기 조정

##### 필터 영역 (Filter Area) - **재디자인**
- **위치**: 헤더 바로 아래, 최소 여백 (`pt-4`)
- **스타일**: 텍스트 기반, 라벨 없음, 배경색 없음, 그림자 제거
- **검색 반경 선택**:
  - 활성: `text-[#2F6E4F] font-semibold`
  - 비활성: `text-[#555555] hover:text-[#1C1C1E]`
  - 간격: `gap-4` (16px)
- **기관 유형 선택**:
  - 활성: Primary 색상 `#2F6E4F` 통일 (기존 타입별 색상 제거)
  - 비활성: `text-[#555555] hover:text-[#1C1C1E]`
  - 간격: `gap-4` (16px)
  - 숫자 제거: "재활스포츠기관 (24개)" → "재활스포츠기관"
- **레이아웃**: 한 줄로 배치, 반응형으로 스크롤 가능
- **Border 중심**: 그림자 대신 Border로 구분

##### 지도 영역 (Map Area)
- 화면의 70% 이상을 차지하는 핵심 영역
- 사용자의 현재 위치를 중심으로 주변 병원들이 핀으로 표시됩니다.
- 필터 변경 시 실시간으로 마커 업데이트

##### 목록 영역 (List Area)
- 지도 좌측 또는 하단에 위치
- 필터링된 병원/재활기관 목록 표시
- 각 항목 클릭 시 상세 정보 Bottom Sheet 표시

## 화면 1-A: 병원 상세 정보

**목표**: 사용자가 가장 필요로 하는 정보(연락처, 위치)와 행동(전화, 길찾기)을 즉시 실행하게 한다.

*(화면 1의 바텀 시트가 확장되거나, 팝업 형태로 부드럽게 올라오는 방식)*

```ascii
+-----------------------------------------+
|                                         |
|               (배경: 지도)                |
|                                         |
+-----------------------------------------+
| [1] 서울리워크병원          [X] 닫기    |
|     정형외과, 재활의학과                  |
+-----------------------------------------+
| 📍 서울특별시 강남구 테헤란로 123        |
| 📞 02-1234-5678                         |
+-----------------------------------------+
|                                         |
|    [2]     📞 전화 걸기                 |
|                                         |
+-----------------------------------------+
|    [3]     🗺️ 길찾기 (외부 앱 연동)       |
|                                         |
+-----------------------------------------+
```

#### 주요 요소 설명

##### 핵심 정보
- 병원명과 주요 진료과를 가장 상단에 크게 표시
- 사용자가 올바른 정보를 보고 있음을 확인시켜 줍니다.

##### 가장 중요한 행동 버튼 (Primary CTA)
- **'전화 걸기'**는 환자와 보호자에게 가장 시급하고 중요한 기능
- 가장 크고 눈에 잘 띄는 버튼으로 제공합니다.

##### 두 번째 행동 버튼 (Secondary CTA)
- '길찾기'는 외부 지도 앱(카카오맵, 네이버 지도 등)으로 연결
- 사용자가 익숙한 방식으로 길을 찾도록 돕습니다.

## 화면 2: 서류 안내

**목표**: 복잡한 서류 목록을 보고 사용자가 압도당하지 않도록, 한 번에 하나의 정보에만 집중하게 만든다.

**반응형 레이아웃**:
- **모바일**: 세로 스크롤, 아코디언 형태로 정보 표시
- **데스크톱**: 2-3 컬럼 그리드로 여러 서류를 동시에 볼 수 있도록 배치

```ascii
+-----------------------------------------+
| [1] 서류 안내                           |
|     차근차근 준비하면 어렵지 않아요.        |
+-----------------------------------------+
| [2] ⚠️ 법률/의료 자문이 아닌 참고용 정보입니다.|
+-----------------------------------------+
| [3] ▼ 요양급여신청서 (치료비/간병비)    |
+-----------------------------------------+
| [4] ▼ 휴업급여청구서 (일 못 한 기간 급여) |
+-----------------------------------------+
|    +----------------------------------+   |
|    | [4-A] AI 요약                    |   |
|    |  💡 이 서류는 아파서 일하지 못한  |   |
|    |     기간의 평균 임금을 신청하는  |   |
|    |     서류입니다.                  |   |
|    +----------------------------------+   |
|    | [4-B] 📄 서식 다운로드           |   |
|    +----------------------------------+   |
|    | [4-C] ✍️ 작성 예시 보기          |   |
|    +----------------------------------+   |
+-----------------------------------------+
```

#### 주요 요소 설명
